Introduction

When a user is attempting to access some service, for example a website, their computer, or a mobile application, they often must gain authorization before using that service. Authentication is performed in two steps. First, the user must identify themselves. Second, they must verify that they are, in fact, the user they claim to be.

Allowing a user to log in only once they have provided the correct password is one extremely common method for performing this verification, but there are many others. In this module we will discuss how passwords are used, what makes some passwords stronger than other passwords, and how attackers might attempt to crack a password.

Module Goals

  • Understand how a password used to verify a user's identity
  • Demonstrate several strategies for cracking passwords
  • Examine how to prevent each strategy
