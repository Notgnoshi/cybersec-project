Conclusion

You've completed the CyberEd module on hashing! Let's take a look at the key points covered in this module:

  • Message digests are used most often to verify data integrity
  • Hash functions are used to make message digests, which characterize an input text
  • Hash functions are one-way functions, which means you cannot convert a digest back to the original message
  • Small changes in the input produce large changes in the output; this is called confusion
  • Patterns in the input do not make patterns in the output; this is called diffusion
  • Variable-length inputs produce fixed-length hashes
  • It's hard to produce a collision

We started by motivating the need for message digests with a story about Alice, Bob, and Chuck, and then learned about what a message digest is, how a hash function makes one, and what makes a hash function good.

After going over the basics, you used a tool to generate message digests for any messages you wanted. If you tested a bunch of different messages, you saw how each hash algorithm generated a digest that was always the same size, and you also saw that even if your message was almost identical to a different message, the digest was completely different.

Finally, you saw how keyed hashing could be used to add a password to your message digest, making it incredibly difficult for someone to fake, even if they had the message you sent and the digest you sent with it.
