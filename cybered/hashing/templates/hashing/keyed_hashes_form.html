Keyed Hashes

In our first example, Alice had to write down her message digest and give it to Bob. As a reminder, that digest was 

5578833a0c6cb26394a1414140718cab
This is kind of random, and it would be easy to get it wrong, don't you think? What if Alice sends long messages like this to Bob all the time? Then every time she has a new message, she has to write down the message digest, meet Bob, and give him the digest.

Instead of doing this process for every single message Alice wants to send, Alice and Bob can use something called a keyed hash. Alice and Bob can meet up, and come up with a secret key, or password, that only they know. Instead of using this key to encrypt the messages (because they are too long), they can use this password to make keyed digests.

Making a keyed digest is very simple. Instead of just creating a digest from her original message, Alice combines the message with her secret key first. Bob also knows the secret key, so when he gets the message, he combines it with the secret key, and then generates the digest, and checks if it's the same, just like before. The difference here is that Alice is allowed to send the message digest along with the message. Even if Chuck intercepts the message and changes it, he doesn't know the secret key, so he can't figure out what the new message digest should be, and he can't trick Bob!
